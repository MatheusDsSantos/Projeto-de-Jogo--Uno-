Visão Geral do Projeto

O que é o projeto?

Este projeto implementa o famoso jogo de cartas UNO em Python, mas com um objetivo além de jogar: demonstrar na prática o funcionamento de quatro estruturas de dados clássicas estudadas em Ciência da Computação.

O jogo pode ser jogado de duas formas: pelo terminal (python uno.py) ou pelo navegador (python servidor.py e abrir http://localhost:8000), onde você enfrenta de 1 a 3 bots. Nas duas versões, quem manda é o mesmo motor de regras em Python.

💡 Por que usar estruturas de dados?

Estruturas de dados são formas organizadas de guardar e manipular informações. Escolher a estrutura certa para cada problema torna o código mais eficiente, mais legível e mais fácil de manter.

As quatro estruturas usadas

📚 Pilha (Stack)

Usada no monte de compra e no monte de descarte. Funciona como uma pilha de pratos: você sempre pega e coloca no topo. Arquivo: estruturas/pilha.py

🔄 Fila (Queue)

Usada para controlar a ordem dos jogadores. Funciona como uma fila de banco: o primeiro que chegou é o primeiro atendido. Arquivo: estruturas/fila.py

🔗 Lista Encadeada (Linked List)

Usada para guardar as cartas na mão de cada jogador. Cada carta aponta para a próxima, formando uma corrente. Arquivo: estruturas/lista_encadeada.py

🌳 Árvore Binária de Busca (BST)

Usada para montar o ranking dos jogadores ordenado pela quantidade de cartas — sem usar sort(). Arquivo: estruturas/arvore.py

Regras do UNO implementadas

  • Cada jogador começa com 7 cartas
  • Só pode jogar uma carta se ela tiver a mesma cor ou o mesmo valor da carta do topo
  • Curingas podem sempre ser jogados e permitem trocar a cor
  • Se não tiver carta válida, o jogador compra uma carta do monte
  • Cartas especiais: Pular, Inverter, +2, Curinga +4
  • Quem ficar sem cartas vence

O módulo random — embaralhando as cartas

O módulo random fornece funções para gerar resultados aleatórios. No projeto, usamos random.shuffle() para embaralhar as cartas. 

import random

cartas = ["Vermelho 1", "Azul 5", "Verde 3"]
print(cartas)
# Saída: ['Vermelho 1', 'Azul 5', 'Verde 3']

random.shuffle(cartas)   # Embaralha a lista no lugar
print(cartas)
# Saída: ['Verde 3', 'Vermelho 1', 'Azul 5']  (ordem aleatória)
✅ Importante

random.shuffle() modifica a lista original diretamente — ela não cria uma nova lista. Por isso passamos self._cartas direto, sem precisar guardar o resultado.

O for repete um bloco de código para cada item de uma coleção. É a forma mais comum de percorrer listas no Python.

deque — fila eficiente do Python

deque (pronuncia-se "dek", de Double-Ended Queue) é uma estrutura de dados que permite adicionar e remover itens eficientemente tanto do início quanto do final. 

from collections import deque

fila = deque()
fila.append("João")    # Adiciona ao final
fila.append("Maria")
fila.append("Pedro")

primeiro = fila.popleft()  # Remove do início: "João"

# Inverter a fila (carta Inverter do UNO)
fila = deque(reversed(fila))
💡 Por que deque e não lista?

Remover o primeiro elemento de uma lista Python é lento (o Python precisa mover todos os outros elementos). O deque faz isso em tempo constante, muito mais eficiente para filas.

Recursão — uma função que chama a si mesma

Recursão é quando uma função chama ela mesma para resolver versões menores do mesmo problema. Toda recursão precisa de um caso base (a condição que faz ela parar) — sem ele, a função chamaria a si mesma para sempre. Você vai precisar desse conceito para entender a Árvore Binária de Busca, pois percorrer uma árvore é naturalmente recursivo: cada filho de um nó é, ele próprio, a raiz de uma árvore menor (uma subárvore). 

# Exemplo clássico: somar os números de 1 até n
def soma_ate(n):
    if n == 0:          # CASO BASE: para de chamar a si mesma
        return 0
    return n + soma_ate(n - 1)   # PASSO RECURSIVO: problema menor

soma_ate(3)
# soma_ate(3) → 3 + soma_ate(2)
#                   → 2 + soma_ate(1)
#                       → 1 + soma_ate(0)
#                           → 0      (caso base!)
# Resultado: 3 + 2 + 1 + 0 = 6
✅ Onde aparece no projeto

Os métodos em_ordem() e altura() da Árvore Binária de Busca (estruturas/arvore.py) usam recursão: o caso base é no is None (chegou numa ponta vazia da árvore), e o passo recursivo é visitar as subárvores esquerda e direita.

Orientação a Objetos (Classes)

O que é uma Classe?

Pense em uma classe como uma planta baixa ou molde. Ela define como um objeto vai ser, quais informações ele guarda e o que ele pode fazer.

Um objeto é uma instância criada a partir dessa classe — é a "casa construída" com base na planta. 

# Classe = molde / planta baixa
class Carta:
    def __init__(self, cor, valor):
        self.cor   = cor
        self.valor = valor

# Objeto = instância criada a partir do molde
carta1 = Carta("Vermelho", "5")   # Cria uma carta
carta2 = Carta("Azul", "Pular")   # Cria outra carta

print(carta1.cor)    # Vermelho
print(carta2.valor)  # Pular

O parâmetro self

self é uma referência ao próprio objeto. Quando você escreve self.cor dentro da classe, você está dizendo: "a variável cor que pertence a este objeto específico". 

class Carta:
    def __init__(self, cor, valor):
        # "self.cor" = atributo do objeto
        # "cor"      = parâmetro recebido
        self.cor   = cor    # guarda a cor NO objeto
        self.valor = valor  # guarda o valor NO objeto

carta_vermelha = Carta("Vermelho", "7")
carta_azul     = Carta("Azul", "3")

# Cada objeto tem seus próprios dados
print(carta_vermelha.cor)  # Vermelho
print(carta_azul.cor)      # Azul
✅ Analogia simples

Pense que self é como dizer "meu" em português. self.cor = "minha cor". Quando você cria dois objetos, cada um tem "sua própria" cor.

Métodos — funções dentro de classes

Métodos são funções que pertencem a uma classe. Eles definem o que o objeto sabe fazer. 

class Carta:
    def __init__(self, cor, valor):
        self.cor   = cor
        self.valor = valor

    # Método especial: chamado quando fazemos print(carta)
    def __str__(self):
        return f"[{self.cor} {self.valor}]"

    # Método normal: verifica se pode jogar
    def pode_jogar_sobre(self, carta_do_topo):
        if self.cor == "Curinga":
            return True
        return self.cor == carta_do_topo.cor or self.valor == carta_do_topo.valor

# Usando os métodos
minha_carta = Carta("Vermelho", "5")
topo        = Carta("Vermelho", "3")

print(minha_carta)                        # [Vermelho 5]
print(minha_carta.pode_jogar_sobre(topo)) # True (mesma cor)

Encadeamento de atributos — self.mao.tamanho

Quando um atributo de um objeto é outro objeto, podemos acessar os atributos e métodos desse segundo objeto usando o ponto . várias vezes. Isso se chama encadeamento. 

# A classe Jogador tem um atributo "mao"
# que é um objeto do tipo ListaEncadeada

class Jogador:
    def __init__(self, nome):
        self.nome = nome
        self.mao  = ListaEncadeada()  # mao é um objeto!

class ListaEncadeada:
    def __init__(self):
        self.tamanho = 0   # tamanho é um atributo de ListaEncadeada

# Como acessar:
jogador = Jogador("Ana")

jogador.nome           # → "Ana"          (atributo direto do Jogador)
jogador.mao            # → objeto ListaEncadeada
jogador.mao.tamanho    # → 0              (atributo do objeto dentro do Jogador)
jogador ──► (objeto Jogador) │ nome = "Ana" │ mao ──► (objeto ListaEncadeada) │ inicio = None │ tamanho = 0 jogador.mao.tamanho │ │ └── atributo de ListaEncadeada │ └────── atributo de Jogador (que é um ListaEncadeada) └────────────── o objeto Jogador

Atributos com underscore — self._cartas

Por convenção, atributos com underscore no início (_cartas) indicam que são privados: devem ser usados apenas dentro da própria classe, não de fora. 

class Pilha:
    def __init__(self):
        self._cartas = []   # "_" = "use apenas dentro da classe"

    def empurrar(self, carta):
        self._cartas.append(carta)  # OK — usamos dentro da classe

# De fora, o certo é usar os métodos, não acessar direto:
pilha = Pilha()
pilha.empurrar(carta)    # ✅ certo
pilha._cartas             # ⚠️ tecnicamente funciona, mas evite

Estruturas de Dados em Profundidade

Pilha (Stack) — LIFO

Uma Pilha é uma estrutura onde o último elemento inserido é o primeiro a ser removido. A sigla LIFO vem do inglês: Last In, First Out (último a entrar, primeiro a sair).

PILHA (Stack) — como uma pilha de pratos: ┌─────────────┐ ◄── TOPO (último inserido, primeiro a sair) │ [Verde 7] │ ◄── empurrar() adiciona aqui │ [Azul 3] │ retirar() remove daqui │ [Verm. 5] │ └─────────────┘ ◄── BASE (primeiro inserido, último a sair) Operações: empurrar(carta) → adiciona no topo retirar() → remove do topo e retorna ver_topo() → olha o topo sem remover
class Pilha:
    def __init__(self):
        self._cartas = []         # Lista Python como base

    def empurrar(self, carta):
        self._cartas.append(carta)  # append() = adiciona ao final (topo)

    def retirar(self):
        return self._cartas.pop()   # pop() = remove e retorna o último

    def ver_topo(self):
        return self._cartas[-1]    # [-1] = último elemento da lista
🃏 No UNO

Monte de compra: você sempre compra a carta do topo. Monte de descarte: você sempre joga sua carta no topo, e a carta visível é o topo da pilha.

Fila (Queue) — FIFO

Uma Fila é uma estrutura onde o primeiro elemento inserido é o primeiro a ser removido. A sigla FIFO vem do inglês: First In, First Out (primeiro a entrar, primeiro a sair).

FILA (Queue) — como uma fila de banco: ENTRADA SAÍDA ▼ ▼ [Pedro] → [Maria] → [João] → [Ana] entrar() proximo() entrar(jogador) → adiciona no final da fila proximo() → remove do início e retorna inverter() → inverte toda a ordem (carta Inverter)
from collections import deque

class Fila:
    def __init__(self):
        self._jogadores = deque()

    def entrar(self, jogador):
        self._jogadores.append(jogador)    # Entra no final

    def proximo(self):
        return self._jogadores.popleft()  # Sai do início

    def inverter(self):
        self._jogadores = deque(reversed(self._jogadores))

Lista Encadeada (Linked List)

Uma Lista Encadeada é uma sequência de nós onde cada nó guarda um dado e um ponteiro para o próximo nó. Diferente de uma lista comum, os elementos não ficam em posições contíguas na memória — cada nó pode estar em qualquer lugar, pois o que os conecta é o ponteiro.

LISTA ENCADEADA: inicio │ ▼ ┌──────────────┬──────┐ ┌──────────────┬──────┐ ┌──────────────┬──────┐ │ carta: │próx │────►│ carta: │próx │────►│ carta: │próx │ │ [Verm. 5] │ │ │ [Azul 3] │ │ │ [Curinga] │ None │ └──────────────┴──────┘ └──────────────┴──────┘ └──────────────┴──────┘ Nó 0 Nó 1 Nó 2 (último)
class No:
    # Um nó: guarda UMA carta + aponta para o próximo nó
    def __init__(self, carta):
        self.carta   = carta  # O dado em si
        self.proximo = None  # Ponteiro para o próximo nó

class ListaEncadeada:
    def __init__(self):
        self.inicio  = None  # Aponta para o primeiro nó
        self.tamanho = 0

    def adicionar(self, carta):
        novo_no = No(carta)        # Cria um novo nó
        if self.inicio is None:
            self.inicio = novo_no  # Lista vazia: novo nó é o primeiro
        else:
            atual = self.inicio
            while atual.proximo is not None:
                atual = atual.proximo  # Anda até o último nó
            atual.proximo = novo_no    # Conecta o novo nó ao final
        self.tamanho += 1

Árvore Binária de Busca (BST) — dados que se ordenam sozinhos

Uma árvore é uma estrutura hierárquica: em vez de cada elemento apontar para um único próximo (como na lista encadeada), cada elemento — chamado de nó — pode apontar para vários filhos. O nó do topo é a raiz, e nós sem filhos são as folhas.

Uma árvore binária é uma árvore em que cada nó tem no máximo dois filhos: um à esquerda e um à direita. E uma Árvore Binária de Busca (ABB, ou BST em inglês — Binary Search Tree) acrescenta uma regra de ouro sobre onde cada valor pode ficar:

🌳 A propriedade da ABB

Para todo nó da árvore: tudo que está na subárvore esquerda tem chave menor que a do nó, e tudo que está na subárvore direita tem chave maior ou igual. Essa regra vale em todos os níveis, não só na raiz.

ÁRVORE BINÁRIA DE BUSCA — vocabulário: ┌─────────┐ raiz ──►│ (5) │ nível 1 └────┬────┘ ┌──────┴──────┐ ┌───▼───┐ ┌───▼───┐ │ (2) │ │ (7) │ nível 2 └───┬───┘ └───────┘ └───┐ folha ▲ ┌───▼───┐ │ (3) │ nível 3 └───────┘ folha menores que 5 ficam à ESQUERDA ◄── (5) ──► maiores ou iguais à DIREITA altura desta árvore = 3 (número de níveis no caminho mais longo)

Por que o UNO precisa de uma árvore?

Para mostrar o ranking: a lista de jogadores ordenada por quem tem menos cartas (quem está mais perto de vencer). A sacada é que a ABB mantém os dados ordenados pela própria forma da árvore — a posição em que cada jogador é inserido já o deixa no lugar certo. Depois, basta ler a árvore na ordem certa e o ranking sai pronto, sem chamar nenhum sort().

No projeto, a chave de cada nó é a quantidade de cartas na mão do jogador, e o valor é o nome dele.

Inserção — passo a passo

Para inserir, compare a nova chave com o nó atual: menor → vai para a esquerda; maior ou igual → vai para a direita. Repita até achar um espaço vazio (None) e pendure o novo nó ali. Veja a construção do ranking de uma partida real, inserindo os jogadores na ordem em que estão na fila — Ana tem 5 cartas, Beto tem 2, Caio tem 7 e Duda tem 2:

inserir(5, "Ana") inserir(2, "Beto") inserir(7, "Caio") árvore vazia: 2 < 5? esquerda: 7 >= 5? direita: vira a raiz (5,Ana) (5,Ana) (5,Ana) / / \ (2,Beto) (2,Beto) (7,Caio) inserir(2, "Duda") → compara com a raiz: 2 < 5, vai para a ESQUERDA compara com (2,Beto): 2 >= 2, vai para a DIREITA (5,Ana) / \ (2,Beto) (7,Caio) \ (2,Duda) ◄── empate (chave igual) vai para a direita
⚠️ Chaves repetidas

Duas pessoas podem ter a mesma quantidade de cartas (como Beto e Duda, com 2). Por isso a regra do projeto é maior OU IGUAL vai para a direita — o empate não se perde, só fica depois do que já estava na árvore.

Percurso em ordem (in-order) — a mágica da ordenação

Existem várias formas de visitar todos os nós de uma árvore (pré-ordem, em ordem, pós-ordem). A que nos interessa é o percurso em ordem, que visita: 1º toda a subárvore esquerda → 2º o próprio nó → 3º toda a subárvore direita, aplicando essa regra recursivamente em cada nó. Pela propriedade da ABB, isso entrega as chaves em ordem crescente:

Percurso EM ORDEM na árvore que montamos: (5,Ana) ④ / \ ② (2,Beto) (7,Caio) ⑤ \ (2,Duda) ③ 1. Desce tudo para a esquerda a partir da raiz... chega em (2,Beto) 2. Visita (2,Beto) → 1º do ranking 3. Direita de Beto: visita (2,Duda) → 2º do ranking 4. Subárvore esquerda da raiz acabou: visita (5,Ana) → 3º 5. Direita da raiz: visita (7,Caio) → 4º Resultado: [(2,Beto), (2,Duda), (5,Ana), (7,Caio)] ▲ ordenado do menor para o maior, sem sort()! No jogo: 1º Beto (2 cartas) · 2º Duda (2) · 3º Ana (5) · 4º Caio (7)

Busca — por que "de busca" no nome

A mesma regra que guia a inserção também guia a busca: para achar uma chave, compare com o nó atual e desça só para um dos lados. A cada passo você descarta metade da árvore (quando ela está equilibrada) — é a mesma ideia da busca binária, daí o nome.

Altura e desempenho

A altura é o número de níveis no caminho mais longo da raiz até uma folha. Ela determina o custo das operações: inserir e buscar custam O(altura).

  • Árvore equilibrada (galhos bem distribuídos): altura ≈ log₂(n) → operações O(log n), muito rápidas.
  • Pior caso: inserindo chaves já ordenadas (1, 2, 3, 4...), todo mundo vai para a direita e a árvore degenera num "espeto" — vira uma lista encadeada de altura n, e as operações caem para O(n).
EQUILIBRADA (boa) DEGENERADA / "espeto" (pior caso) inserindo 4, 2, 6, 1, 3: inserindo 1, 2, 3, 4 (já ordenado): (4) (1) / \ \ (2) (6) (2) / \ \ (1) (3) (3) \ altura = 3 ≈ log₂(5) (4) altura = 4 = n
🃏 No UNO

Com 2 a 4 jogadores a árvore é minúscula, então desempenho não é o ponto — o objetivo é didático: mostrar que a ABB ordena "de graça" pela sua própria estrutura. Em problemas reais (índices de bancos de dados, por exemplo), essa mesma ideia, com árvores que se reequilibram sozinhas (AVL, Red-Black), é o que torna buscas em milhões de registros quase instantâneas.

Comparação entre as quatro estruturas

Estrutura Forma Entra Sai / Lê No UNO
Pilha Linear (LIFO) No topo Do topo Monte de compra e descarte
Fila Linear (FIFO) No final Do início Ordem dos jogadores
Lista Encadeada Linear (nó a nó) No final De qualquer posição Cartas na mão de cada jogador
Árvore Binária de Busca Hierárquica Na posição ditada pela chave Em ordem (já ordenado) Ranking dos jogadores
✅ Como decorar a diferença

Pilha = pilha de pratos (mexe só no topo). Fila = fila de banco (entra atrás, sai na frente). Lista encadeada = corrente de elos (anda elo por elo). ABB = árvore genealógica organizada (menores à esquerda, maiores à direita — e ler "da esquerda para a direita" devolve tudo em ordem).

As Classes da Pasta estruturas/ Explicadas Linha a Linha

Toda estrutura de dados do projeto vive na pasta estruturas/, cada uma em seu próprio arquivo, independente das regras do UNO — elas guardam "cartas" e "jogadores", mas funcionariam igual guardando qualquer outra coisa. Esta é a parte mais importante do projeto para estudar.

Classes No e ListaEncadeada — estruturas/lista_encadeada.py

No é a unidade mínima da lista. ListaEncadeada gerencia o conjunto de nós.

class No:
    def __init__(self, carta):
        self.carta   = carta  # A carta guardada neste nó
        self.proximo = None  # Ponteiro: aponta para o próximo nó (ou None se for o último)

class ListaEncadeada:
    def __init__(self):
        self.inicio  = None  # Primeiro nó da lista
        self.tamanho = 0    # Conta quantas cartas há na mão

    def adicionar(self, carta):
        novo_no = No(carta)
        if self.inicio is None:     # Se a lista estiver vazia
            self.inicio = novo_no
        else:
            atual = self.inicio       # Começa do início
            while atual.proximo is not None:
                atual = atual.proximo  # Avança até o último nó
            atual.proximo = novo_no    # Conecta o novo nó
        self.tamanho += 1

    def remover_por_indice(self, indice):
        if indice == 0:                      # Remove o primeiro
            carta = self.inicio.carta
            self.inicio = self.inicio.proximo # O segundo vira o primeiro
            self.tamanho -= 1
            return carta
        atual = self.inicio
        for _ in range(indice - 1):        # Chega até o nó anterior
            atual = atual.proximo
        carta = atual.proximo.carta
        atual.proximo = atual.proximo.proximo  # "Pula" o nó removido
        self.tamanho -= 1
        return carta
💡 Como funciona a remoção

Remover um nó do meio é como remover um elo de uma corrente: você conecta o elo anterior diretamente ao elo seguinte, e o elo do meio "cai fora". O Python descarta automaticamente objetos sem referências (garbage collection).

Removendo o nó 1 (índice 1): ANTES: [Verm.5] → [Azul 3] → [Curinga] 0 1 2 atual aponta para nó 0: atual.proximo = atual.proximo.proximo ↕ ↕ [Azul 3] [Curinga] DEPOIS: [Verm.5] → [Curinga] [Azul 3] (descartado) 0 2

Classe Pilha — estruturas/pilha.py

class Pilha:
    def __init__(self):
        self._cartas = []

    def empurrar(self, carta):
        self._cartas.append(carta)    # Topo = final da lista

    def retirar(self):
        if self.esta_vazia(): return None
        return self._cartas.pop()     # Remove e retorna o último

    def ver_topo(self):
        if self.esta_vazia(): return None
        return self._cartas[-1]       # Olha sem remover

    def esta_vazia(self):
        return len(self._cartas) == 0

    def tamanho(self):
        return len(self._cartas)

    def embaralhar(self):
        random.shuffle(self._cartas)

Classe Fila — estruturas/fila.py

class Fila:
    def __init__(self):
        self._jogadores = deque()

    def entrar(self, jogador):
        self._jogadores.append(jogador)      # Entra no final

    def proximo(self):
        return self._jogadores.popleft()    # Sai do início

    def inverter(self):
        # reversed() inverte o deque, depois criamos um novo deque
        self._jogadores = deque(reversed(self._jogadores))

    def esta_vazia(self):
        return len(self._jogadores) == 0

Classes NoArvore e ArvoreBinariaBusca — estruturas/arvore.py

Assim como a lista encadeada tem o No, a árvore tem o NoArvore. A diferença é que, em vez de um ponteiro (proximo), cada nó da árvore tem dois (esquerda e direita) — e guarda um par chave + valor em vez de um dado só: 

class NoArvore:
    def __init__(self, chave, valor):
        self.chave    = chave   # Quantidade de cartas (usada para comparar)
        self.valor    = valor   # Nome do jogador (o dado que carregamos junto)
        self.esquerda = None   # Filho com chave MENOR
        self.direita  = None   # Filho com chave MAIOR OU IGUAL

Método inserir() — descendo até achar um lugar vago

A inserção é feita com um while True: a partir da raiz, compara a chave nova com a do nó atual e desce para o lado correto. Quando o lado escolhido está vazio (None), pendura o nó novo ali e encerra com return: 

    def inserir(self, chave, valor):
        novo = NoArvore(chave, valor)

        if self.raiz is None:       # Árvore vazia:
            self.raiz = novo           # o novo nó vira a raiz
            return

        atual = self.raiz             # Começa comparando com a raiz
        while True:
            if chave < atual.chave:             # MENOR → esquerda
                if atual.esquerda is None:      # Achou lugar vago?
                    atual.esquerda = novo          # Pendura aqui
                    return
                atual = atual.esquerda             # Senão, continua descendo
            else:                               # MAIOR OU IGUAL → direita
                if atual.direita is None:
                    atual.direita = novo
                    return
                atual = atual.direita

Método em_ordem() — leitura ordenada com recursão

Aqui entra a recursão da Seção 2. O método público só prepara a lista de resultado e dispara o método recursivo a partir da raiz. O recursivo segue a receita do percurso em ordem: esquerda → o próprio nó → direita: 

    def em_ordem(self):
        resultado = []
        self._em_ordem_recursivo(self.raiz, resultado)
        return resultado    # Lista de tuplas (chave, valor) em ordem crescente

    def _em_ordem_recursivo(self, no, resultado):
        if no is None:      # CASO BASE: ponta vazia, volta sem fazer nada
            return
        self._em_ordem_recursivo(no.esquerda, resultado)  # 1º: tudo à esquerda
        resultado.append((no.chave, no.valor))            # 2º: o próprio nó
        self._em_ordem_recursivo(no.direita, resultado)   # 3º: tudo à direita
✅ Para acompanhar a recursão

Leia assim: "antes de me anotar na lista, anote todo mundo menor que eu (subárvore esquerda); depois de mim, anote todo mundo maior ou igual (subárvore direita)". Como cada nó aplica essa mesma regra, a lista final sai inteira em ordem crescente.

Métodos buscar() e altura()

    def buscar(self, chave):
        atual = self.raiz
        while atual is not None:
            if chave == atual.chave:     # Achou!
                return atual.valor
            if chave < atual.chave:      # Menor: só pode estar à esquerda
                atual = atual.esquerda
            else:                        # Maior: só pode estar à direita
                atual = atual.direita
        return None                      # Chegou numa ponta vazia: não existe

    def altura(self):
        return self._altura_recursiva(self.raiz)

    def _altura_recursiva(self, no):
        if no is None:               # CASO BASE: árvore vazia tem altura 0
            return 0
        altura_esq = self._altura_recursiva(no.esquerda)
        altura_dir = self._altura_recursiva(no.direita)
        return 1 + max(altura_esq, altura_dir)  # 1 (eu) + o maior galho

Onde a árvore é usada no jogo — JogoUno.ranking()

O motor do jogo (modelo/jogo.py) monta uma árvore nova a cada consulta do ranking: insere cada jogador usando a quantidade de cartas como chave e lê tudo com em_ordem(). O resultado já sai ordenado de quem está ganhando para quem está perdendo: 

    def ranking(self):
        arvore = ArvoreBinariaBusca()
        for jogador in self.fila_jogadores.listar():
            arvore.inserir(jogador.quantidade_cartas(), jogador.nome)

        return [
            {"nome": nome, "cartas": quantidade}
            for quantidade, nome in arvore.em_ordem()
        ]

Na versão web, o painel "🏆 Ranking" da lateral é exatamente o retorno de ranking(), atualizado a cada jogada. No terminal, o ranking aparece ao final da partida.

Fluxo Completo do Jogo

Todo o fluxo abaixo acontece dentro do motor do jogo, em modelo/jogo.py (classe JogoUno). Repare como cada passo é, no fundo, uma operação sobre uma das estruturas da Seção 4.

Inicialização

1
Criar o baralho

108 cartas são criadas e embaralhadas. Todas são colocadas na Pilha de compra.

2
Criar jogadores

Cada jogador é criado e entra na Fila na ordem que foram cadastrados.

3
Distribuir 7 cartas

Cada carta é retirada da Pilha de compra e adicionada à Lista Encadeada de cada jogador.

4
Virar a primeira carta

Uma carta (não-curinga) é retirada da Pilha de compra e colocada na Pilha de descarte. A cor atual é definida.

Loop principal — cada rodada

A
Pegar o jogador da vez

fila_jogadores.proximo() retira o primeiro da Fila.

B
Verificar se tem carta válida

Percorre a Lista Encadeada do jogador comparando cada carta com o topo da Pilha de descarte.

C
Jogar ou comprar

Se tem carta válida: jogador escolhe qual jogar, a carta é removida da Lista Encadeada e empurrada na Pilha de descarte. Se não tem: compra uma da Pilha de compra.

D
Verificar vitória

Se a Lista Encadeada do jogador estiver vazia (tamanho == 0), ele venceu.

E
Devolver para a fila

O jogador entra no final da Fila novamente com fila_jogadores.entrar(jogador).

F
Aplicar efeitos especiais

Se a carta jogada foi especial, aplica o efeito sobre o próximo da Fila.

G
Atualizar o ranking

A interface chama ranking(), que insere cada jogador em uma Árvore Binária de Busca (chave = quantidade de cartas) e lê o resultado com o percurso em ordem.

Efeitos das cartas especiais

CartaEfeitoEstrutura envolvida
Pular O próximo jogador é removido da fila, devolvido ao final sem jogar. Fila — proximo() + entrar()
Inverter A ordem de toda a fila é invertida. Fila — inverter()
+2 O próximo compra 2 cartas da pilha e perde a vez. Pilha + Lista Encadeada + Fila
Curinga Jogador atual escolhe a nova cor vigente. Atributo cor_atual do JogoUno
+4 Próximo compra 4 cartas, perde a vez e o atual escolhe a cor. Pilha + Lista Encadeada + Fila

Reciclagem do monte

Quando a Pilha de compra fica vazia, o método _reciclar_descarte() é chamado: 

    def _reciclar_descarte(self):
        topo = self.monte_descarte.retirar()  # Guarda a carta do topo

        # Move todas as outras cartas do descarte para o monte
        while not self.monte_descarte.esta_vazia():
            self.monte_compra.empurrar(self.monte_descarte.retirar())

        self.monte_compra.embaralhar()        # Embaralha o novo monte
        self.monte_descarte.empurrar(topo)   # Devolve o topo ao descarte

Por que cada estrutura foi escolhida?

Pilha para os montes de cartas

O monte de compra e o de descarte são naturalmente pilhas: você sempre interage com a carta do topo. Quando compra, pega do topo. Quando descarta, coloca no topo. Nunca precisa acessar cartas do meio. A Pilha é perfeita para isso — operações de topo são instantâneas.

Fila para a ordem dos jogadores

A ordem dos jogadores segue exatamente a lógica FIFO: quem jogou primeiro, vai jogar por último na próxima rodada. Após jogar, o jogador vai para o final da fila. A Fila captura isso perfeitamente. A carta Inverter também é elegante: basta inverter a fila inteira.

Lista Encadeada para a mão dos jogadores

A mão de um jogador muda constantemente: recebe cartas novas (distribuição, compra) e remove cartas específicas (quando joga). A Lista Encadeada permite remover qualquer carta de qualquer posição de forma eficiente, sem precisar reorganizar toda a estrutura como seria necessário em uma lista comum.

Árvore Binária de Busca para o ranking

O ranking precisa dos jogadores ordenados pela quantidade de cartas. Em vez de ordenar uma lista com sort(), cada jogador é inserido em uma ABB usando a quantidade de cartas como chave — a forma da árvore já guarda a ordem. O percurso em ordem devolve o ranking pronto: quem tem menos cartas primeiro. É o exemplo perfeito de estrutura cuja organização interna substitui um algoritmo de ordenação.

📌 O(1) vs O(n)

O(1) significa que a operação leva o mesmo tempo independente do tamanho da estrutura (ideal). O(n) significa que o tempo cresce proporcionalmente ao tamanho — se dobrar os dados, dobra o tempo. Para o UNO com poucos jogadores e poucas cartas, isso não é um problema na prática.

💡 A ideia central da arquitetura

O motor (modelo/jogo.py) contém todas as regras mas não usa input() nem print(). Assim, o mesmo motor serve as duas interfaces: o terminal conversa com ele por chamadas de método, e o navegador conversa por requisições HTTP. Separar "regras" de "tela" é uma prática clássica de organização de software.

modelo/ — as regras do UNO em poucas palavras

Carta (carta.py)

Guarda cor e valor. O método pode_jogar_sobre() decide se a jogada é válida: curinga sempre pode; senão, precisa bater a cor ou o valor.

Jogador (jogador.py)

Tem um nome, uma flag eh_bot e a mão de cartas — que é uma Lista Encadeada. Receber e jogar carta delegam para a lista.

JogoUno (jogo.py)

O motor: cria as 108 cartas na Pilha de compra, distribui 7 a cada jogador, controla os turnos pela Fila, aplica Pular/Inverter/+2/+4 e monta o ranking com a ABB.

bot (bot.py)

Decide a jogada do computador: joga uma carta comum válida se tiver (guarda curingas para o aperto) e, ao jogar curinga, escolhe a cor que mais tem na mão.

servidor.py — o backend web

Usa só a biblioteca padrão do Python (http.server) — não precisa instalar nada. Ele serve os arquivos da pasta frontend/ e expõe uma API JSON que a página usa para conversar com o motor:

RotaO que faz
POST /api/novo-jogoCria uma partida: você + 1 a 3 bots
GET /api/estadoDevolve o estado: sua mão, topo do descarte, ranking (ABB), histórico...
POST /api/jogarVocê joga a carta de um índice (e a cor, se for curinga)
POST /api/comprarVocê compra 1 carta e passa a vez
POST /api/bot-jogarExecuta uma jogada do bot da vez (a página chama uma por vez, com pausa)

frontend/ — a tela do jogo

HTML, CSS e JavaScript puros, sem frameworks. O script.js chama a API com fetch(), redesenha a mesa a cada resposta e abre um seletor de cores quando você joga um curinga. O painel lateral mostra o ranking vindo da ABB e o histórico de jogadas. Importante: o JavaScript não conhece as regras do UNO — toda validação acontece no backend Python.
