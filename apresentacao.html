Apresentação do Projeto

O que é o projeto

  • Um jogo de UNO completo e jogável, feito em Python puro.
  • Só usa a biblioteca padrão — sem instalar nada (pip não é preciso).
  • Objetivo didático: mostrar 4 estruturas de dados funcionando num caso real, não só na teoria.
  • Duas formas de jogar: terminal e navegador.

As 4 Estruturas de Dados

Todas implementadas do zero — o coração do projeto.

📚 Pilha (LIFO)

Monte de compra e monte de descarte. A última carta jogada fica no topo.

estruturas/pilha.py

🚶 Fila (FIFO)

Ordem dos turnos. Quem joga vai para o fim; o inverter muda o sentido.

estruturas/fila.py

🔗 Lista Encadeada

A mão de cartas de cada jogador. Adiciona e remove carta por índice.

estruturas/lista_encadeada.py

🌳 Árvore Binária de Busca

Monta o ranking ordenado sem usar sort().

estruturas/arvore.py

Destaque: o Ranking pela Árvore

  • A cada jogada, cada jogador é inserido na árvore usando a quantidade de cartas como chave.
  • O percurso em ordem (esquerda → raiz → direita) já devolve todos ordenados.
  • Do que tem menos cartas (ganhando) ao que tem mais (perdendo).
  • Tudo isso sem chamar sort() — a estrutura faz o trabalho.

inserir(3, "Matheus")
inserir(1, "Bot Ana")
inserir(7, "Bot Beto")

em_ordem() →
[(1,"Bot Ana"), (3,"Matheus"), (7,"Bot Beto")]

Como está organizado

O motor do jogo tem todas as regras, mas não faz input() nem print(). As interfaces conversam com o usuário — a mesma lógica serve para as duas.

Navegadorfrontend/
→
Servidorservidor.py (API JSON)
→
Motor do Jogomodelo/jogo.py
→
Estruturaspilha · fila · lista · árvore

O terminal (uno.py) liga direto no mesmo motor.

O que o jogo tem

  • Regras completas: números, Pular, Inverter, +2, Curinga e +4.
  • Aviso de "UNO!" e condição de vitória.
  • Baralho de 108 cartas.
  • Bots: jogam sozinhos, guardam curingas para o fim e escolhem a cor mais frequente na mão.
  • Botões (?) na tela explicam qual estrutura está em uso em cada parte.
  • Ranking ao vivo no painel lateral.
